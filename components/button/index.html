Button

Description

With the Button component you can render a button element. The style variant and size of the button can be added with the variant prop and is composed like variant.size.

Properties

Property
Type
Default

variant (optional)

primary, secondary, ghost, text

primary

size (optional)

small, large

large

disabled (optional)

boolean

space (optional)

ResponsiveStyleValue<string>

none

Import

import { Button } from '@marigold/components';


Demo

Live-Code Example

<Button
  variant="secondary"
  size="small"
  onClick={() => {
    alert('Hello!');
  }}
>
  Secondary
</Button>

Usage

Primary Button

The primary button function is to achieve the most common or most important goal of the user, it has more visual weight than the secondary options. There should not be more than one primary button in a screen at the time.


Default


Disabled


Secondary Button

Secondary buttons are an alternative to the user to the Primary option. They are used for actions like canceling, skipping, resetting, declining an offer etc.


Default


Disabled


Ghost Button

They are used for miscellaneous actions, they are important but is not the ones where it is desired that the user focus at the moment.


Default


Disabled


Suplementary Text Button

They can be used in a similar way of a secondary button, but they are meant to be less relevant to the user.


Default


Disabled


Button with icon

Icons can be added in Order to make clearer what a button does. Buttons should not be made only with icons, because every user may understand them on a different way.



Small Buttons

If there are many actions in one page, is it possible to use small buttons instead of big ones. As well if the page is required to be open on mobile devices.

Usage

Do

Use a ghost button next to a Primary button.

Do

Use a Text button next to a Primary button.

Do

Use a Text button next to another text button.

Don't

Use two Primary buttons on the same page, there should be only one function that calls the most attention.

Don't

Use two Secondary buttons next to each other, it loses hierarchie and is confusing.

Don't

Mix normal sized buttons with small buttons inmediatly next or below each other.
